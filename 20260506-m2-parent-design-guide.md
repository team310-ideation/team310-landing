# M2 학부모 타겟 랜딩페이지 디자인 가이드

> 목적: M1 정적 랜딩을 "학부모·학생 인터뷰 전환"에 더 강한 M2 랜딩으로 개선하기 위한 디자인·UX 기준.
> 결론: 학부모를 "디지털에 약한 부모 세대"로 보지 않는다. 핵심 타겟은 바쁘지만 검색·메신저·동영상·커뮤니티를 능숙하게 쓰는 40~50대 고관여 의사결정자다.

## 2. M1 디자인 가이드 점검

### 유지할 것

```text
✓ 딥 네이비 + 오렌지 CTA + 따뜻한 화이트는 신뢰·행동·온기를 잘 만든다.
✓ 모바일 우선, 본문 16px 이상, 한국어 line-height 1.7 이상은 유지한다.
✓ 담당자 실명, 전화번호, 이메일, 개인정보 처리 설명은 전환 장벽을 낮춘다.
✓ "문제 공감 → 팀 신뢰 → 접근 방식 → 멘토/ASM → 신청" 흐름은 맞다.
✓ 문화상품권, 30분, 대면/비대면 가능 메시지는 반복 노출한다.
```

### 수정할 것

```text
△ "부모 세대 친화 = 트렌디하지 않게"라는 표현은 너무 거칠다.
  → 최신 학부모는 트렌드를 모르는 사람이 아니라, 바쁜 의사결정자다.

△ Hero가 감성 문장 중심이면 5초 안에 문제 매칭이 약하다.
  → 첫 화면에서 "생기부 / 예비고1 / 담임상담 / 30분 인터뷰"가 보여야 한다.

△ 텍스트 카드가 길면 학부모가 카카오 인앱에서 스캔하기 어렵다.
  → M2는 카드보다 탭·칩·짧은 질문형 블록을 우선한다.

△ "AI"를 전면에 세우면 50대에서 이해·신뢰 장벽이 생길 수 있다.
  → AI는 후순위 설명. 전면 메시지는 "내 아이 기록 기준 우선순위"다.

△ 팀/멘토 사진 필수라고만 적혀 있고 실제 대체 전략이 없다.
  → 실제 사진이 없으면 이니셜 아바타보다 팀 활동 사진, 회의 사진, 데모 캡처가 낫다.

△ 카카오 공유만으로는 부족하다.
  → 네이버 검색·네이버 카페·밴드 공유까지 고려한 OG/SEO/문장 구조가 필요하다.
```

## 3. M2 디자인 원칙

### 1) 최신감은 애니메이션이 아니라 "반응성"에서 만든다

M2는 React/Vite + Framer Motion을 권장한다. 다만 애니메이션은 브랜드 과시가 아니라 이해를 돕는 수준이어야 한다.

```text
권장:
✓ Hero 문구 1회 fade-up, 180~260ms
✓ 문제 칩 선택 시 부드러운 전환
✓ 숫자 지표 카운트업 1회
✓ FAQ accordion height transition
✓ 스크롤 reveal은 한 번만, 이동량 12~20px 이하

금지:
✕ 계속 움직이는 배경
✕ 과한 parallax
✕ 긴 로딩 애니메이션
✕ CTA 주변을 흔드는 애니메이션
✕ reduced-motion 미지원
```

### 2) 학부모는 "구매자"이기 전에 "불안한 보호자"다

첫 화면에서 바로 팔거나 기술을 설명하지 않는다. 보호자 입장에서 가장 먼저 필요한 것은 "내 아이 정보가 안전한가", "이 사람들이 믿을 만한가", "내 고민을 제대로 이해하는가"다.

```text
M2 Hero 첫 화면 필수 요소:
1. Literal H1: 학부모·학생 인터뷰 모집
2. 문제 문장: "아이 기록 기준으로 지금 뭘 해야 할지 막막하신가요?"
3. 신뢰 칩: ASM 17기 / 100+ 인터뷰 / 자료 업로드 없이 가능
4. CTA: 30분 인터뷰 신청하기
5. 보조 CTA: 어떤 이야기를 듣나요?
```

### 3) AI는 "마법"이 아니라 "정리 도구"로 말한다

40대는 정보검색·업무용 AI에 상대적으로 익숙할 수 있지만, 50대는 인지도와 실제 사용 사이의 간극이 있다. 따라서 AI-first 카피는 피한다.

```text
나쁜 표현:
AI가 우리 아이 입시 전략을 자동으로 만들어드립니다.

좋은 표현:
사람이 먼저 듣고, AI는 누적된 기록과 고민을 정리해 다음 질문을 놓치지 않게 돕습니다.
```

### 4) "공식성"보다 "구체성"이 신뢰를 만든다

정부 지원 프로젝트, 멘토, 숫자는 좋지만 단독으로는 광고처럼 보일 수 있다. M2에서는 신뢰 신호를 구체적인 책임 구조로 바꾼다.

```text
신뢰 스택:
✓ 담당자 실명과 직접 연락처
✓ 인터뷰에서 묻는 것 / 묻지 않는 것
✓ 녹음 동의 여부 선택
✓ 자료 업로드 필수 아님
✓ 외부 공개 시 비식별화
✓ ASM 공식 링크
✓ 팀원 실제 사진 또는 활동 사진
```

### 5) 카카오·네이버·밴드에서 읽히는 문장으로 설계한다

학부모 유입은 카카오 공유와 네이버 검색·카페·블로그 맥락이 강하다. 인앱 브라우저에서 스크롤을 오래 하지 않아도 핵심이 보여야 한다.

```text
OG title:
학부모·학생 30분 인터뷰 모집 | Team 310

OG description:
생기부·예비고1·담임상담 준비가 막막한 학부모님의 이야기를 듣습니다. 자료 업로드 없이 가능, 문화상품권 제공.

네이버 검색용 첫 문단:
Team 310은 AI·SW마에스트로 17기 프로젝트 팀으로, 중3·예비고1 학부모와 학생의 입시·진로 고민을 듣는 30분 인터뷰를 진행합니다.
```

## 4. M2 정보 구조

```text
1. Hero
   - 학부모·학생 인터뷰 모집
   - 3대 공감 질문: 생기부 / 예비고1 / 담임상담
   - 30분 CTA + 문화상품권 + 자료 업로드 없이 가능

2. Pain Match
   - "어떤 고민에 가까우신가요?"
   - 탭 또는 segmented control 5개
   - 각 탭은 2문장 + 인터뷰에서 듣고 싶은 것

3. What Happens in Interview
   - 1) 현재 고민 듣기
   - 2) 기록/상황 맥락 정리
   - 3) 우리가 이해한 문제가 맞는지 확인
   - 녹음 선택, 자료 업로드 선택, 중단 가능

4. Why Team 310
   - 팀원 3명, 실명, 경험, 한 줄 이유
   - 교육 서비스 운영 경험 / KOPLE / AI·Data 근거
   - 실제 사진 또는 활동 사진 우선

5. How We Work
   - 사람 먼저 듣기 → 가설 검증 → 맥락 구조화 → 작게 검증
   - "하지 않을 일"과 "집중할 일"을 나란히 배치

6. Trust
   - ASM 17기, 멘토 네트워크, 개인정보 처리 원칙
   - 숫자보다 책임 문장 중심

7. Apply
   - 짧은 폼
   - 5개 페인포인트 체크박스
   - 자녀 학년, 인터뷰 방식, 편한 시간, 연락처

8. FAQ
   - 개인정보
   - 기획심의 활용 여부
   - 서비스 판매 의도
   - 다단계/종교 아님
   - 기관 협력
```

## 5. M2 비주얼 방향

### 전체 톤

```text
키워드:
현대적, 차분함, 공식적, 사람 중심, 과하지 않은 에듀테크

피해야 할 톤:
입시 학원 광고, 초등 학습지 광고, 과한 SaaS 그라디언트, AI 마법사 느낌, 불안 자극형 카피
```

### 컬러 팔레트

M1 팔레트를 유지하되, M2에서는 색의 역할을 더 선명하게 나눈다.

```text
Primary Navy:      #1E3A5F  신뢰, 제목, 헤더, 공식성
Deep Navy:         #102035  Footer, sticky CTA, 강한 대비
Warm White:        #FFFDF9  기본 배경
Surface:           #F7F8F9  문제/FAQ 섹션 배경
CTA Orange:        #F97316  신청 버튼, 전환 포인트
Sage Green:        #4A7C59  개인정보 안심, 완료, 체크
Clear Blue:        #2563EB  링크, 공식 자료, 정보 강조
Soft Yellow:       #FEF3C7  문화상품권/인센티브 보조 배경
Text Charcoal:     #1F2937
Text Secondary:    #5F6B7A
Border:            #E5E7EB
```

원칙:
- Hero는 navy 기반이 맞지만, 배경 이미지는 실제 교육/상담/노트/학교 맥락이 보여야 한다.
- 전체 페이지가 navy·cream만 반복되면 다소 무겁다. 문제 섹션에는 blue, 안심 섹션에는 sage, 인센티브에는 soft yellow를 얕게 쓴다.
- 보라색·남색 그라디언트 중심의 SaaS 톤은 피한다.

### 이미지 전략

```text
우선순위:
1. 팀원 실제 사진
2. 팀 회의/인터뷰 준비/화이트보드 사진
3. 서비스 데모 또는 생기부 맥락을 추상화한 실제 UI 캡처
4. 현실감 있는 교육 현장 사진
5. 이니셜 아바타

금지:
✕ 어두운 스톡 사진
✕ 외국 교실 이미지
✕ 손만 보이는 과한 감성 이미지
✕ AI로 만든 실제 인물처럼 보이는 팀 사진
```

### 타이포그래피

```text
Font: Pretendard Variable
Hero H1: 32px mobile / 48~52px desktop
Section H2: 24px mobile / 34~36px desktop
Body: 16~18px
Caption: 14px
Line-height: 1.55 for headings, 1.7~1.8 for body
Letter-spacing: 0
```

### 레이아웃

```text
Mobile:
- CTA는 화면 하단 sticky 유지
- Hero CTA는 1열
- 문제 선택은 horizontal chip 또는 2열 compact grid
- 팀원/멘토는 카드 1열
- 긴 설명은 accordion/details로 접기

Desktop:
- 최대 폭 1120~1180px
- Hero는 full-bleed image overlay 또는 split이 아닌 텍스트 중심 overlay
- 문제 섹션은 탭 + 상세 패널
- 팀/멘토는 3열 카드
```

## 6. 인터랙션 가이드

### React M2 권장 스택

```text
Vite + React
Framer Motion
lucide-react
CSS Modules 또는 plain CSS
Vercel static build
```

### Motion 규칙

```text
duration: 180~260ms
easing: cubic-bezier(0.2, 0.8, 0.2, 1)
distance: 12~20px
stagger: 최대 60ms
reduced motion: 필수 지원
```

### 컴포넌트

```text
HeroProblemRotator:
- "생기부", "예비고1", "담임상담", "특목·자사"를 3초마다 바꾸되 reduced-motion에서는 정지

PainSegment:
- 5개 페인포인트를 segmented control로 선택
- 선택된 문제에 맞는 학부모 발화와 인터뷰 질문 표시

InterviewProcess:
- 3-step timeline
```
